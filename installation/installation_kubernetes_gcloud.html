Deploying Susi with Kubernetes on Google Cloud¶

  1. Go to Google Cloud Free Trial and sign up. You will get 300 dollars credit for 3 months.
  1. Go to the console page: Console
  1. If you don’t have any projects, a page that would guide you on how to start a project will pop up. Give your project a name.
  1. In the search bar above on console page, enter ‘Compute Engine’ and click on Compute Engine.
  1. If you have not activated billing for compute engine, click enable billing and choose your account to activate your Compute Engine. Your Google Compute Engine should be activated within a few minutes.
  2. Select your computer zone region: gcloud config set compute/zone us-central1-b
  3. You can view your defauls/configure by this command: gcloud config list
  4. In the Web Console, enter:
sudo docker pull jyothiraditya/susi_server
  1. Create a cluster

gcloud container clusters create example-cluster

  1. Ensure kubectl has authentication credentials:

gcloud auth application-default login

  1. query docker images . Copy or remember this id, we will need this id to tag the image.

kubectl run susi_server --image= docker-id --port=8080

  1. Expose the container

kubectl expose deployment susi_server --type="LoadBalancer"

  1. Copy the external IP address for the hello-node app by typing this command.

kubectl get service susi_server

Wait for while and reenter query.Enter your assigned IP address with port 8080 (address:8080) into your browser to check if it is working.
